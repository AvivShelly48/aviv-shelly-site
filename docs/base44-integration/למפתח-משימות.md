# מה צריך לבצע — חיבור תוכנת הניהול לאתר (Base44)

**למי מיועד:** הסוכן/מפתח שמנהל את תוכנת הניהול (אפליקציית Base44).
**המטרה בשורה אחת:** שכשנפתח פרויקט חדש בתוכנת הניהול הוא יופיע אוטומטית באתר, וכשמסמנים "אוכלס" / "נמכר" — האתר יתעדכן לבד.

> כל הפרטים הטכניים המדויקים (פורמט JSON, שמות שדות, טבלאות המרה) נמצאים בשני הקבצים באותה תיקייה:
> **`PUBLIC-API.md`** (איך האתר קורא נתונים) ו-**`SYNC-BRIEF.md`** (איך הניהול דוחף נתונים). המסמך הזה הוא רק **רשימת המשימות** ומפנה אליהם.

---

## התהליך בתמונה אחת
```
[תוכנת הניהול]  ──(1) דוחפת פרויקט/דירה──▶  [אפליקציית Base44 של האתר]  ──(2) האתר קורא──▶  הגולש
   פותחים פרויקט,                                Project / Unit                        גריד פרויקטים
   מסמנים "אוכלס"/"נמכר"                                                                 + לוח דירות זמינות
```

## ✅ צ'קליסט — 4 משימות

### משימה 1 — להקים אפליקציית Base44 חדשה לאתר
אפליקציה **נפרדת** מתוכנת הניהול, עם 3 ישויות (הסכמות המדויקות בקבצים `Project.schema.json`, `Unit.schema.json`, `Lead.schema.json`):
- `Project` — פרויקט (כולל שדה `external_id`)
- `Unit` — דירה (כולל `external_id` ו-`status`: available/reserved/sold)
- `Lead` — פנייה מהטופס

### משימה 2 — לבנות 3 פונקציות ציבוריות (Functions)
בפורמט המדויק שמתואר ב-**`PUBLIC-API.md`** (כולל CORS `Access-Control-Allow-Origin`):
| פונקציה | סוג | תפקיד |
|---|---|---|
| `getPublicProjects` | GET ציבורי | מחזיר את רשימת הפרויקטים לאתר |
| `getPublicUnits` | GET ציבורי | מחזיר את הדירות ללוח הזמינות |
| `submitLead` | POST | קולט פניות מהטופס ושומר `Lead` |

### משימה 3 — לבנות 2 אוטומציות שדוחפות מהניהול לאתר
זה הלב של מה שאביב שאל עליו — התיאור המלא ב-**`SYNC-BRIEF.md`** §2:
- **אוטומציה א' — פרויקט נוצר/עודכן:** על יצירה/עדכון פרויקט בניהול → `POST/PATCH` לישות `Project` באתר, מזוהה לפי `external_id` (upsert). ← *זה מה שגורם לפרויקט חדש להופיע אוטומטית.*
- **אוטומציה ב' — סטטוס משתנה:** על שינוי סטטוס פרויקט/דירה → עדכון `stage`/`status` באתר. ← *זה מה שגורם ל"אוכלס" ול"נמכר" לעדכן את האתר.*
  - פרויקט `אוכלס` → `stage: 3` (פס ההתקדמות עובר ל"אכלוס").
  - דירה `נמכר` → `status: "sold"` (הדירה מתעמעמת/ננעלת בלוח).
  - טבלאות ההמרה המלאות (`category→cat`, `status→sale/stage`) ב-`SYNC-BRIEF.md` §3.

### משימה 4 — למסור לי מזהה אחד
בסיום, שלחו לי את **`APP_ID`** (או ה-URL המלא של בסיס הפונקציות). אני מדביק שורה אחת באתר:
```html
<script>window.SHELLY_BASE44_API = "https://app.base44.com/api/apps/<APP_ID>/functions";</script>
```
ומאותו רגע האתר קורא נתונים חיים.

## מה שנשאר לי לקבל מכם (כדי לסגור את הצד שלכם)
מ-`SYNC-BRIEF.md` §5 — שני פרטים על הדירות:
1. שם ה-entity של הדירה בתוכנת הניהול, שם שדה הסטטוס וערכיו (נמכר/פנוי), ומזהה הפרויקט המשויך.
2. האם תוכנת הניהול תומכת באוטומציה עם HTTP/Webhook יוצא? (אם לא — נתאם חלופה: משיכה מתוזמנת.)

## בשורה אחת לאביב
כל עוד 4 המשימות לא בוצעו — האתר מציג **נתוני דוגמה** ולא מתעדכן לבד.
ברגע שהסוכן מסיים ואני מדביק את ה-`APP_ID` — **פרויקט חדש מופיע אוטומטית, וסימון "אוכלס"/"נמכר" מתעדכן באתר לבד.** אין עבודה כפולה.
